# 项目文件标注说明

> 每个文件的作用，一句话说明。

---

## 根目录

| 文件 | 作用 |
|------|------|
| `app.py` | Flask 主程序，定义所有页面路由和 API 接口，接收前端请求 → 调用 `core/` 算法 → 返回 JSON 结果 |
| `README.md` | 项目说明文档，包含安装和启动方法 |

---

## models/ — 数据模型

| 文件 | 作用 |
|------|------|
| `models/process.py` | 定义进程数据类（Process），包含到达时间、服务时间、周转时间等字段，供调度模块使用 |

---

## core/scheduler/ — 模块1：处理器调度

| 文件 | 作用 |
|------|------|
| `core/scheduler/fcfs.py` | 先来先服务（FCFS）：按到达时间排序，依次执行，非抢占式 |
| `core/scheduler/rr.py` | 时间片轮转（RR）：维护就绪队列，每个进程每次运行一个时间片，未完成则重新入队 |
| `core/scheduler/sjf.py` | 短进程优先（抢占式/SRTF）：每1个时间单位重新选剩余时间最短的进程执行 |
| `core/scheduler/hrn.py` | 最高响应比优先（HRN）：非抢占式，每次选响应比 R=(等待+服务)/服务 最大的进程 |

---

## core/producer_consumer/ — 模块2：生产者-消费者

| 文件 | 作用 |
|------|------|
| `core/producer_consumer/producer_consumer.py` | 多线程同步模拟：使用 mutex + empty/full 信号量 + consume_lock 实现多生产者/多消费者，消费者可连续取 n 个产品 |

---

## core/banker/ — 模块3：银行家算法

| 文件 | 作用 |
|------|------|
| `core/banker/banker.py` | 银行家算法类：包含安全性检查（is_safe_state）和资源请求检查（request_resources），判断系统是否安全、请求是否可分配 |

---

## core/paging/ — 模块4：页面置换

| 文件 | 作用 |
|------|------|
| `core/paging/fifo.py` | 先进先出（FIFO）：淘汰最早进入内存的页面，用 deque 记录进入顺序 |
| `core/paging/lru.py` | 最近最久未使用（LRU）：淘汰 last_used 最小的页面，用字典记录每次访问时间 |
| `core/paging/opt.py` | 最优置换（OPT）：淘汰未来最远使用或永不使用的页面，作为理论基准 |

---

## core/disk/ — 模块5：磁盘调度

| 文件 | 作用 |
|------|------|
| `core/disk/sstf.py` | 最短寻道时间优先（SSTF）：每次选距离当前磁头最近的请求 |
| `core/disk/scan.py` | 电梯算法（SCAN）：磁头沿一个方向移动到最边缘再折返，解决 SSTF 的饥饿问题 |

---

## utils/ — 工具类

| 文件 | 作用 |
|------|------|
| `utils/paging_result.py` | 页面置换结果封装类：记录每一步的内存状态、缺页标记、被淘汰页面 |

---

## templates/ — 前端页面

| 文件 | 作用 |
|------|------|
| `templates/index.html` | 首页：5 个模块卡片导航，点击跳转到各功能页面 |
| `templates/scheduler.html` | 调度页面：编辑进程数据、选择算法、运行模拟、显示甘特图和结果表 |
| `templates/producer_consumer.html` | 生产者-消费者页面：设置参数、运行模拟、实时显示缓冲区格子和事件日志 |
| `templates/banker.html` | 银行家页面：输入资源/进程矩阵、检查安全状态、模拟资源请求 |
| `templates/page_replacement.html` | 页面置换页面：输入引用串和块数、运行模拟、显示内存状态变化矩阵和缺页率 |
| `templates/disk_scheduling.html` | 磁盘调度页面：输入磁道序列和初始位置、运行模拟、显示 SVG 磁头移动轨迹图 |
